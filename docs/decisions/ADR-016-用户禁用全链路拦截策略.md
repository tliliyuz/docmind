# ADR-016: 用户禁用全链路拦截策略

**状态**：已采纳（2026-06-13）
**决策者**：开发团队
**关联文档**：ARCHITECTURE.md §9b.4-§9b.6 / API.md §7

## 上下文

Admin 禁用用户后，需要确保已登录用户被即时拦截，包括已持有的 access_token 和 refresh_token。

## 决策

三层拦截：
1. **`get_current_user()` 依赖**（非 AuthMiddleware）：校验 `user.status`，禁用用户返回 E5010。保持中间件无状态
2. **`login()` / `refresh()` 拦截**：拒绝 disabled 用户登录/刷新
3. **`change_user_status(disabled)` 自动吊销**：调用 `revoke_all_user_tokens()` 清除全部 refresh_token

`revoke_all_user_tokens()` 从 `_` 私有提升为公开函数，admin_service 复用。

## 理由

- 中间件保持无状态（不做 DB 查询），性能优先
- 依赖层校验覆盖所有 API 调用，access_token 15 分钟自然过期
- 禁用时立即吊销 refresh_token，防止用户刷新续期
- Token 吊销复用现有基础设施，零重复代码

## 后果

- `get_current_user()` 改为 async，增加 DB 查询
- E5010 错误码需加入前端 401 拦截器白名单（不踢下线）
