# ADR-004: 知识库可见性弱混合模式

**状态**：已采纳（2026-05-22）
**决策者**：开发团队
**关联文档**：PRD.md §5 / ARCHITECTURE.md §7.6 / API.md §9

## 上下文

当前权限模型为「仅 owner 能看自己的 KB + admin 全局只读」，与 PRD 描述的跨部门知识共享场景不匹配。

## 决策

采用**弱混合模式**：`visibility` 控制 READ（`private` 仅 owner + admin 可读，`public` 所有用户可检索），`ownership` 控制 WRITE（仅 owner 可上传文档；admin 拥有管理级 WRITE——可删除 KB/文档、修正元数据，但不上传文档）。

两步校验不可合并：先判断 visibility（能否读），再判断 ownership（能否写）。

## 理由

- 一个字段解决核心矛盾，不做 ACL/多租户
- Admin 不是「只读管理」：能看一切、能删能改元数据、能清理违规内容，但不能替别人上传文档
- 与 PRD §5.4 权限矩阵对齐

## 后果

- `knowledge_bases` 新增 `visibility ENUM('private','public') DEFAULT 'private'`
- 新增 `GET /api/knowledge-bases/public` 端点
- 所有 KB 接口必须先判断 visibility 再判断 ownership
