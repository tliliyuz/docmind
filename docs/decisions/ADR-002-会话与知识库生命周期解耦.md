# ADR-002: 会话与知识库生命周期解耦

**状态**：已采纳（2026-06-13）
**决策者**：开发团队
**关联文档**：RAG_PIPELINE.md §1 / API.md §5 / DATABASE.md §2.5

## 上下文

两个关联 Bug：
1. 删除 KB 后，`ON DELETE SET NULL` 级联刷新 `updated_at`，旧对话浮到列表顶部
2. KB 删除后会话保留但无提示，用户看到「请先选择知识库」不知原因

## 决策

1. **`last_message_at` 独立排序字段**：仅在 `send_message`/`assistant_reply` 时更新，不复用 `updated_at`
2. **孤儿会话保留**：不删除历史对话，增加前端 Banner 提示与交互引导
3. **`kb_status` 动态计算**：API 层 LEFT JOIN 实时判断 `active`/`deleted`/`unavailable`，不持久化

## 理由

- `updated_at` 语义错乱：标题修改/pin/归档/FK 级联等非消息操作均会污染排序
- 删除会话会让用户认为聊天记录丢失
- `kb_status` 持久化引入同步开销，实时 JOIN 成本可忽略

## 后果

- 新增 Alembic 迁移：`last_message_at` 列 + 复合索引
- 前端新增孤儿会话 Banner、Sidebar 状态标签
